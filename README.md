This project demonstrates the design and implementation of an Online Book Store Database using PostgreSQL. It includes database creation, table design, data import, and SQL queries to analyze book sales, customer information, inventory, and order details.

✨ Features
Designed a relational database with normalized tables.
Created tables for:
Books
Customers
Orders
Imported data using CSV files.
Implemented SQL queries for:
Data retrieval using SELECT, WHERE, ORDER BY, LIMIT
Aggregate functions (SUM, AVG, COUNT)
Joins between multiple tables
GROUP BY and HAVING
Sales and revenue analysis
Inventory management
Customer purchase analysis
Genre and author-based reports
🛠️ Technologies Used
PostgreSQL
SQL
CSV Data Import
📊 Database Schema
Books – Stores book details such as title, author, genre, price, stock, and publication year.
Customers – Stores customer information including name, email, phone, city, and country.
Orders – Stores order records linking customers with purchased books.
📈 Sample SQL Analyses
Retrieve books by genre.
Find books published after a specific year.
Calculate total inventory stock.
Identify the most expensive book.
Compute total revenue generated.
Find the most frequently ordered book.
Analyze books sold by genre and author.
Identify top-spending customers.
Generate city-wise customer purchase reports.
